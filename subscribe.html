Why I Built This

I got really tired of the fact that creating really useful tools was often easier than saving them.

It's way too hard to store and share the digital tools we build during actual work. I'm constantly trying to solve that problem. This isn't the final solution — but it's a damn good patch in the meantime.

After spending an absurd amount of time building in Blender Geometry Nodes, at some point my tools got good enough that I needed a real system. Not a folder of .blend files scattered across Downloads. Not a Gumroad page that breaks my flow every time I need an update. Something that actually behaved like a professional, always-within-reach, perfectly-synced palette of tools.

I couldn't find one. So I built one.

What you're subscribing to isn't a static library. It's a live duplicate of the tools I actually use — the same node groups, the same parametric systems, the same print-ready geometry generators that run in my studio. When I improve something, your copy improves. When I build something new for a project and add it to the library, you get it automatically.

Most of the time you shouldn't even know this is happening. All you should really notice is that every once in a while, your NO3D Tools asset library is gradually getting bigger, more polished, and more packed with info.

Once the add-on is installed in your Blender environment, you'll never need to visit my website to download an update.

No dojo_bool_v2_FINAL_use_this_one.blend in your Downloads folder.

How The Sync Works

When I finish a tool, it goes into the pipeline — metadata synced, .blend file pushed to CDN, version flag updated on the server.

Your addon checks on Blender startup. If something's new, it pulls it down and registers it in your Asset Browser automatically.
